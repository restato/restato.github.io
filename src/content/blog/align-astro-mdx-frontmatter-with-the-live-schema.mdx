생성된 MDX를 실제 Astro 콘텐츠 컬렉션 스키마에 맞춰 게시 전에 검증합니다.

## 문제와 원인

콘텐츠 생성기와 실제 컬렉션 스키마가 서로 다른 게시 날짜 필드 이름을 사용했습니다.

## 해결 방법

src/content/config.ts를 기준으로 date 필드를 생성하고 게시 전에 Astro 프로덕션 빌드를 실행합니다.

## 적용할 수 있는 조건

Astro 콘텐츠 컬렉션이 생성된 MDX 프런트매터를 거부하거나 여러 작성 도구가 같은 컬렉션을 사용할 때 적용합니다.

## 적용하면 안 되는 조건

해당 필드를 읽는 모든 페이지와 컴포넌트를 확인하지 않은 상태에서는 운영 스키마 필드명을 바꾸지 않습니다.

## 검증

공개 블로그 저장소에서 스키마 계약 테스트와 npm run build를 실행합니다.

생성된 지식 문서가 현재 콘텐츠 스키마로 프로덕션 빌드를 통과했습니다.
