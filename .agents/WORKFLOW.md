# Restato Content OS Workflow

## 실행 모드

### 1. On-demand
사용자가 주제를 주면 즉시 조사하고 작성·검토·발행한다. 하루 발행 수 제한은 없다.

### 2. Trend watch
관심 주제의 공식 발표, 릴리스, 중요한 실무 글을 주기적으로 확인한다. 의미 있는 변화가 있을 때만 글을 작성한다.

### 3. Project log
최근 커밋과 기능 변경에서 재사용 가능한 배움이 있으면 개발기로 작성한다.

### 4. Evergreen
검색 의도가 분명하고 장기적으로 도움이 되는 가이드를 작성한다.

### 5. Digest
여러 소식이 각각 단독 글감으로 부족하지만 함께 보면 가치가 있을 때 주간·주제별 정리 글로 묶는다.

## 파이프라인
1. `trend-finder`: 후보 발굴 또는 사용자 주제 정규화
2. `researcher`: 공식 출처·저장소·실제 코드 조사
3. `planner`: 독자와 검색 의도에 맞는 개요 설계
4. `writer`: Restato 문체의 MDX 초안 작성
5. `seo`: 제목·설명·slug·내부 링크 점검
6. `illustrator`: 필요한 시각 자료 계획 또는 생성
7. `reviewer`: 사실성·중복·가독성·기술 정확성 검수
8. `publisher`: 파일 생성, 빌드 확인, 커밋 및 배포
9. memory 갱신: `published.md`, `ideas.md`, `keywords.md`

## 판단 규칙
- 즉시 요청은 예약 작업보다 우선한다.
- 발행 횟수가 아니라 품질과 중요도로 판단한다.
- 기존 글과 의도가 같으면 업데이트한다.
- 새로운 정보가 없으면 발행하지 않는다.
- 리뷰 점수는 참고 지표이며 사실 오류·빌드 실패가 있으면 점수와 무관하게 발행 금지다.
