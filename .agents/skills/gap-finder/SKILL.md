# Gap Finder

## 목적
이미 많이 다뤄진 주제를 반복하지 않고, 독자가 실제로 궁금해하지만 기존 검색 결과가 충분히 답하지 못한 질문을 찾는다.

## 절차
1. 주제의 핵심 검색 의도 3~7개를 정의한다.
2. 공식 문서, 상위 검색 결과, GitHub 이슈·토론, 개발자 커뮤니티에서 반복되는 질문을 수집한다.
3. 기존 Restato 글의 제목, 설명, 본문 관점을 비교한다.
4. 아래 유형의 공백을 표시한다.
   - 설치법은 많지만 실제 운영 사례가 없음
   - 기능 소개는 많지만 마이그레이션 방법이 없음
   - 성공 사례는 많지만 실패 조건과 제한이 없음
   - 영문 자료는 있으나 한국어 실전 예제가 없음
   - 단순 예제는 많지만 테스트·배포·비용까지 이어지지 않음
5. 새 글, 기존 글 업데이트, 시리즈 편입 중 하나를 선택한다.

## 출력
```yaml
topic: ""
primary_intent: ""
existing_coverage:
  - ""
gaps:
  - question: ""
    evidence: ""
    opportunity: ""
recommended_action: new-post | update-existing | series | skip
recommended_title: ""
unique_promise: "이 글만 읽으면 독자가 무엇을 할 수 있는가"
```

## 발행 조건
- 공백이 실제 근거로 확인되어야 한다.
- '한국어 글이 적다'만으로는 충분하지 않다.
- 최소 한 개의 실행 가능한 예제, 비교표, 체크리스트 또는 실패 사례를 제공할 수 있어야 한다.
- 검색 결과를 그대로 요약하지 말고 공식 자료와 직접 검증으로 보완한다.
