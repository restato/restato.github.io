# Scoring Engine

## 목적
발행 후보를 감이 아니라 일관된 기준으로 평가한다. 자동 발행의 기본 임계값은 85점이며, 사용자가 직접 지정한 주제는 점수와 무관하게 조사하되 품질 기준은 동일하게 적용한다.

## 입력
- 주제명
- 공식 출처와 발표일
- 기존 글 목록
- 예상 독자
- 직접 테스트 또는 코드 예제 가능 여부
- 검색 결과에서 발견한 콘텐츠 공백

## 점수표 (100점)
- 시의성 15: 최근 공식 발표·릴리스와 직접 관련
- 독자 가치 20: 문제 해결이나 의사결정에 실질적 도움
- 블로그 적합도 15: AI, 개발, 자동화, Vibe Coding, 실제 프로젝트와의 연관성
- 독창성 15: 기존 글과 다른 관점이나 새 실험
- 검색 지속성 10: 일회성 뉴스가 아닌 후속 검색 가능성
- 실증 가능성 15: 공식 문서, 코드, 저장소, 테스트로 검증 가능
- 콘텐츠 공백 10: 상위 문서가 다루지 않는 질문을 해결

## 감점
- 공식 출처가 없음: -30
- 기존 글과 검색 의도가 사실상 동일: -25
- 직접 확인하지 않은 성능·비용·사용 경험을 단정: 발행 금지
- 제목만 바꾼 뉴스 요약: -20
- 독자 행동으로 이어질 예제나 결론이 없음: -10

## 판정
- 90~100: 우선 발행
- 85~89: 발행 가능
- 70~84: ideas.md에 보류하고 추가 근거 수집
- 0~69: 제외

## 출력 형식
```yaml
topic: ""
score: 0
breakdown:
  timeliness: 0
  reader_value: 0
  fit: 0
  originality: 0
  evergreen: 0
  evidence: 0
  content_gap: 0
decision: publish | hold | reject | update-existing
reason: ""
recommended_angle: ""
```

## 규칙
점수는 근거를 대신하지 않는다. 수치를 추정해 사실처럼 표현하지 말고, 각 점수의 이유를 한 문장 이상 기록한다.
