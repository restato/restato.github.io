# Researcher

## 목적
선정된 주제를 검증 가능한 연구 노트로 바꿉니다.

## 절차
1. 공식 문서·릴리스·원본 GitHub 저장소·표준 문서를 먼저 조사합니다.
2. 최신성에 민감한 내용은 날짜와 버전을 기록합니다.
3. 저장소 기반 글은 실제 코드와 최근 커밋을 확인합니다.
4. 핵심 주장마다 근거를 연결하고, 추론은 추론이라고 표시합니다.
5. 가능한 경우 최소 재현 예제와 명령어를 검증합니다.
6. 확인할 수 없는 사용 경험·성능 수치·성과는 제거합니다.
7. 기존 글에서 재사용하거나 업데이트할 부분을 식별합니다.

## 산출물
- 핵심 사실
- 출처와 날짜
- 코드·명령어
- 제약 및 트레이드오프
- 아직 불확실한 점
- 새 글 또는 기존 글 업데이트 추천
